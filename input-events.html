This specification defines additions to events for text and related input. This specification builds on the Document Object Model Events Level 3.

Introduction

This document describes editing related additions to 2 events - Input and beforeinput which are described in the UI events spec. The goal of these events is to allow authors to understand and/or override default edit behavior both before and after editing occurs.

Problem and Use Cases

The problems solved by and use cases for this document by this document are tracked in the Editing Explainer document.

Input Event Types

Input events are sent before (beforeinput event) and after (input event) a user attempts to edit the markup. This includes characters, deletions, and other related edits.

Interface InputEvent

readonly attribute DOMString inputType

inputType holds the type of input. Allowed values include insertText, replaceText, replaceContent, insertNewline, deleteContent, deleteComposedCharacter, undo, redo. During initialization, the user agent is responsible for populating this field as follows:

  1. If the user is inserting plain text using a keyboard, IME, speech, or similar method, the inputType MUST be insertText.
    1. Set the value of data to be the plain text string to be inserted.
    2. Set the value of targetRanges to NULL.
  2. If the user indicates replacement of existing content using a keyboard, IME, speech, or similar method, the inputType MUST be replaceContent.
    1. Set the value of data to be a document fragment containing the new content that is to be inserted.
    2. Set the value of targetRanges to ranges that cover the content that is to be replaced.
  3. If the user is inserting a line break using a keyboard, IME, speech, or similar method, the inputType MUST be insertLineBreak
    1. Set the value of data to be NULL.
    2. Set the value of targetRanges to be NULL.
  4. If the user is inserting a paragraph break using a keyboard, IME, speech, or similar method, the inputType MUST be insertParagraph
    1. Set the value of data to be NULL.
    2. Set the value of targetRanges to be NULL.
  5. If the user indicated deletion of a character during character compisition using IME, the inputType MUST be deleteComposedCharacter
    1. Set the value of data to be "forward" or "backward", depending on whether the caret will move forward or backward in connection with the deletion of the character.
    2. Set the value of targetRanges to NULL.
    3. Set the value of cancelable to false
  6. If the user indicated deletion of content that causes the caret to move using a keyboard, speech, or similar method, the inputType MUST be deleteContent
    1. Set the value of data to be "forward" or "backward", depending on whether the caret would move forward or backward in connection with the deletion of content.
    2. Set the value of targetRanges to NULL.
  7. If the user indicates undo using a keyboard, IME, speech, or similar method, the inputType MUST be undo
    1. Set the value of data to be NULL.
    2. Set the value of targetRanges to NULL.
  8. If the user indicates redo using a keyboard, IME, speech, or similar method, the inputType MUST be redo
    1. Set the value of data to be NULL.
    2. Set the value of targetRanges to NULL.

The un-initialized value of this attribute MUST be "" (the empty string).

readonly attribute Ranges targetRanges

The ranges in the document that are affected by this event, unless the affected ranges are related to the current selection.

The Input event types are listed below.

beforeinput
Type beforeinput
Interface InputEvent
Sync / Async Sync
Bubbles Yes
Trusted Targets Any Element with contenteditable attribute enabled.
Cancelable Varies: False for inputTypes insertText and deleteComposedCharacter during IME composition. True otherwise
Default action Varies: 'Update the DOM' for contentEditable=typing editing hosts for inputTypes insertText and deleteComposedCharacter. 'Update the DOM element' for contentEditable=true editing hosts for all inputTypes. None otherwise.
Context
(trusted events)
  • Event.target: event target that is about to be updated
  • UIEvent.view: Window
  • UIEvent.detail: 0
  • InputEvent.data: the string containing the data that will be added to the element, which MAY be the empty string if the content has been deleted, or NULL if it doesn't apply.
  • InputEvent.isComposing: true if this event is dispatched during a dead key sequence or while an input method editor is active (such that composition events are being dispatched); false otherwise.
  • InputEvent.cancelable: true if this event can be canceled. false otherwise.

    All beforeinput events MUST be cancelable, unless they take place during IME composition.

A user agent MUST dispatch this event when the user has attempted to input in a contenteditable element. It does not necessarily mean the user agent will then update the DOM.

A user agent MUST NOT dispatch this event due to events that are not caused by attempted user input, such as system events.

input
Type input
Interface InputEvent
Sync / Async Sync
Bubbles Yes
Trusted Targets Any Element with contenteditable attribute enabled.
Cancelable No
Default action None
Context
(trusted events)
  • Event.target: event target that was just updated
  • UIEvent.view: Window
  • UIEvent.detail: 0
  • InputEvent.data: the string containing the data that was added to the element, which MAY be the empty string if the content has been deleted, or NULL if it doesn't apply.
  • InputEvent.isComposing: true if this event is dispatched during a dead key sequence or while an input method editor is active (such that composition events are being dispatched); false otherwise.
  • InputEvent.cancelable: false

A user agent MUST dispatch this event immediately after the DOM has been updated.
