contentEditableDisabled allows the disabling of certain editing actions on contenteditable editing hosts.

The DOMCommandTokenList

           interface DOMCommandTokenList : DOMTokenList {
             [SameObject] readonly attribute DOMString supported;
           };

The supported attribute returns the supported tokens as a string.

Extensions to the ElementContentEditable mixin

           partial interface ElementContentEditable {
             [SameObject, PutForwards=value] readonly attribute DOMCommandTokenList contentEditableDisabled;
           };

The contentEditableDisabled attribute, when specified on an element that is a contenteditable host, restricts the types of editing operations available by means of user agent native user interfaces and keyboard shortcuts handlers. Its value must be an unordered set of unique space-separated tokens [[HTML]] that are ASCII case-insensitive [[INFRA]]. The allowed values are:

  • history: makes the element not be part of any browser-provided undo stack.
  • inserHorizontalRule
  • unorderedList
  • orderedList
  • insertLink
  • insertLineBreak
  • insertParagraph
  • formatBold
  • formatItalic
  • formatUnderline
  • formatStrikeThrough
  • formatSuperscript
  • formatSubscript
  • formatJustifyFull
  • formatJustifyCenter
  • formatJustifyRight
  • formatJustifyLeft
  • formatJustify: combines formatJustifyFull, formatJustifyCenter, formatJustifyRight and formatJustifyLeft
  • formatIndent
  • formatOutdent
  • formatDent: combines formatIndent and formatOutdent
  • formatSetBlockTextDirection
  • formatSetInlineTextDirection
  • formatFontColor
  • formatBackColor
  • formatFontName

Unless specified, each specified token input controls that would otherwise trigger input events with Input Types [[input-events-1]] of the same value.
